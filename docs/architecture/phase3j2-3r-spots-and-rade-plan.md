## Pre-port template (use for every upstream port task)

For tasks marked `[PORT]`, run this checklist BEFORE reading the upstream source:

1. State the upstream file: `<path/to/upstream/file>`
2. State the NereusSDR target: `<path/to/nereus/file>`
3. Check provenance: `grep -l "<nereus-path>" docs/attribution/<UPSTREAM>-PROVENANCE.md`. If empty, the port is a new attribution event.
4. Plan: if (3) returned nothing, add upstream header byte-for-byte AND a PROVENANCE row in the same commit.
5. Inline cite format: `// From <upstream> <path>:<line> [vX.Y.Z|@sha]`.
6. Preserve all `//AUTHOR_TAG` and behavioral comments verbatim. Position-adjust with `[original inline comment from <file>:<line>]` if restructuring moves code.

`scripts/verify-thetis-headers.py` (existing) and `scripts/verify-freedv-headers.py` (new, see Phase A) enforce mechanically.

## Per-task TDD discipline (NereusSDR convention)

Every task follows this loop, even when the task is a port:

1. Write a failing test for the public surface this task adds.
2. Run the new test 2x. Expect FAIL.
3. Implement the minimal code.
4. Run the new test 2x. Expect PASS.
5. Commit GPG-signed.

Per memory rule: **DO NOT** run the full ctest suite per task. Only NEW tests during the task. Pre-commit hooks run the verifiers. Full suite runs once at end-of-epic (Phase N).

## Commit message convention

`<type>(<scope>): <description>` matching existing NereusSDR commits. Example:
- `feat(spots): add DxClusterClient with login + spot parsing`
- `feat(rade): RadeChannel skeleton with start/stop/lifecycle`
- `port(freedv): vendored radae_nopy + libopus, CMake glue`
- `attrib(freedv-gui): FREEDV-GUI-PROVENANCE.md scaffold + scripts`
- `docs(3j2-3r): bench verification matrices`

Always end commits with:
```
Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>
```

GPG-signed. Never `--no-gpg-sign`.

- [ ] **Step 1: Extract DxSpot struct to its own header**

The original `SpotCollectorClient.h` includes `DxClusterClient.h` for `DxSpot`. We want this struct standalone so other clients (and tests) can include it without pulling DX cluster code.

Create `src/core/DxSpot.h`:

```cpp
// From AetherSDR src/core/DxClusterClient.h:13-23 [@0cd4559]
//
// Copyright (c) 2026 Jeremy / KK7GWY
// Licensed under MIT (per AetherSDR LICENSE)
//
// Modification history (NereusSDR)
//   Ported 2026-MM-DD by JJ Boyd / KG4VCF
//   AI tooling: Claude Code
//
// Extracted from DxClusterClient.h so multiple clients can share the type
// without pulling in DX cluster code.
#pragma once

#include <QString>
#include <QTime>

namespace NereusSDR {

struct DxSpot {
    QString spotterCall;     // W3LPL
    double  freqMhz{0.0};    // 14.025 (converted from kHz)
    QString dxCall;          // JA1ABC
    QString comment;         // "CW big signal"
    QTime   utcTime;         // 18:24 UTC
    QString source;          // "Cluster", "RBN", "WSJT-X", "SpotCollector", "POTA", "FreeDV", "PSK"
    QString color;           // #AARRGGBB for radio spot color (optional)
    int     snr{0};          // signal-to-noise ratio (dB)
```

### Task B5: FreeDVReporterClient (Engine.IO/Socket.IO WebSocket)

**Phase:** 3J-2
**[PORT]** freedv-gui `src/reporting/FreeDVReporter.{h,cpp}` [@77e793a] - this is a NEW port from freedv-gui (replaces AetherSDR's lacking FreeDvClient).

This is the most architecturally important port: it's what makes "FreeDV Reporter" actually useful in NereusSDR. It must:

- Open WebSocket to `wss://qso.freedv.org/socket.io/?EIO=4&transport=websocket`
- Handle Engine.IO Open (type '0') by parsing `pingInterval` and starting ping timer
- Handle Engine.IO Ping (type '2') by replying with Pong (type '3')
- Handle Engine.IO Pong (type '3') silently
- Handle Engine.IO Message (type '4') by dispatching to Socket.IO layer
- Handle Socket.IO Connect ACK (type '0' inside Engine.IO Message)
- Handle Socket.IO Event (type '2') by parsing JSON array `["event_name", {...}]`
- Dispatch events: `new_connection` / `freq_change` / `tx_report` / `rx_report` / `remove_connection` / `bulk_update` / `message_update` / `connection_successful`
- For each event, update internal `QHash<QString sid, FreeDVStation>` AND emit appropriate signals
- For `freq_change`: ALSO emit `spotReceived(DxSpot)` synthesized from station state (this is the dual-feed per design doc Section 4 Flow 2)
- For `rx_report`: ALSO emit `spotReceived(DxSpot)` for the reported transmitter
- Auto-reconnect with exponential backoff on disconnect
- Send Socket.IO Connect with view-only auth payload `{"role":"view","protocol_version":2}`

The DSP/protocol logic comes from freedv-gui [@77e793a]. Cross-reference inline cites point at the `freedv-gui/src/reporting/FreeDVReporter.cpp` line that originated each method.
