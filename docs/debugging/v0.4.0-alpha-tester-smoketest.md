# NereusSDR v0.4.0 Alpha Test: OpenHPSDR Radios (P1 & P2)

**Hi, and thank you.** This is the v0.4.0 alpha-test guide. v0.4.0 is a
substantial minor release on top of v0.3.2 with five pieces of work landing
together: **3M-4 PureSignal** (the big new feature: feedback DDC, calcc/IQC
engine, PsForm and AmpView dialogs, two-tone IMD overlay, enabled across
the ANAN family plus HL2 and Hermes), a **Display + DSP-Options refactor**
(Setup → DSP page, Thetis-faithful FFT slider with 7 windows, NF-aware
grid, peak detector, multimeter page, in-place filter resize), the
**3M-3a-iv anti-VOX cancellation feed** (closes the v0.3.2 gap so the
Anti-VOX gain control actually works), **live-apply sample rate** (no
disconnect needed when switching rates), and the **AF Gain + VAX
calibration fix** (KM4BLG closed a long-standing distortion bug; VAX bus
stays calibrated for digital-mode apps).

If you ran the v0.3.2 alpha test, you don't need to redo it. The v0.3.2 doc
([docs/debugging/v0.3.2-alpha-tester-smoketest.md](v0.3.2-alpha-tester-smoketest.md))
is still the right reference for DEXP/VOX baseline, the HL2 mi0bot RF/Tune
slider rework, the PA Setup pages, and the PA-cal hotfix verification.
This v0.4.0 doc focuses only on what changed.

---

## What's new in v0.4.0 (the short list)

1. **PureSignal works end-to-end.** Tools → PureSignal opens the new
   PsForm dialog (port of Thetis PsForm). The two-tone IMD overlay on
   the spectrum lets you watch IM3 and IM5 in real time. AmpView opens
   from PsForm's btnPSAmpView. Bottom-banner FB+PS indicator pair shows
   feedback-channel and correction status. Enabled on every supported
   ANAN P1 and P2 SKU plus Hermes Lite 2 and plain Hermes (HL2 has its
   own negative-ATT path, AutoAtt convergence guard, ATT-on-TX master
   force-enable, and psSampleRate=0 sentinel resolution).
2. **The whole Display + DSP-Options page lands.** Setup → DSP gains a
   full Thetis-faithful 18-control page with warning icons,
   time-to-last-change readout, and an RX/TX combo split. Filter Impulse
   Cache load/save, in-place filter resize, per-mode buffer/filter
   live-apply. SettingsSchemaVersion v5 migrates Buffer/Filter Size to
   per-direction automatically. The spectrum gains a Thetis-faithful FFT
   slider with 7 windows + live bin width, K-based auto-zoom, NF-aware
   grid (auto-track noise floor), per-band NF priming, Hz/bin auto-zoom
   override, a new SpectrumPeaksPage with PeakBlobDetector + Active Peak
   Hold, and a Multimeter page.
3. **Anti-VOX gain slider actually does something now.** v0.3.2 had the
   slider plumbed but the RX-audio feedback path into DEXP was never
   connected, so the slider had no effect. v0.4.0 wires
   RxDspWorker → TxWorkerThread → TxChannel::sendAntiVoxData per chunk.
   Setup → Transmit → DEXP/VOX gains the full grpAntiVOX trio (Enable,
   Gain, Tau ms).
4. **Sample rate change without reconnect.** The Radio Info combo on
   Setup → Hardware no longer requires a disconnect/reconnect cycle when
   you flip rates. 12-step Thetis-faithful coordinator routes through
   WDSP's SetXcmInrate path instead of destroying and recreating the
   channel. HL2 P1 now offers 48 / 96 / 192 / 384 kHz (was missing 384;
   matches mi0bot exactly).
5. **AF Gain audio quality + VAX calibration.** AF slider routes through
   WDSP SetRXAPanelGain1 (PR #218 from **KM4BLG**) instead of fighting
   the master-volume atomic post-DSP. Closes a long-standing distortion
   bug where the panel.gain1=4.0 default was leaking +12 dB silently.
   VAX bus inverse-scales by 1/afGain so digital-mode apps (WSJT-X,
   FT8, FLDIGI) stay calibrated regardless of speaker AF slider position.

## Things to know before you key up

**PureSignal calibration is bench-test territory.** PS works but
calibration takes some patience. Plan on 10-30 seconds of two-tone runtime
to get a clean correction (longer on HL2 due to the smaller feedback
path). The AmpView traces and the IMD overlay are your eyeballs. K2GX
(ANAN-8000DLE) is the primary bench reference; field reports especially
welcome on radios we cannot bench locally.

**HL2 PureSignal is supported but constrained.** HL2's feedback DDC has
less dynamic range than the bigger ANAN boards. We landed negative-ATT
support, AutoAtt convergence, and the psSampleRate=0 sentinel to make HL2
PS run at all. If your HL2 PS calibration converges, please tell us;
if it gets stuck or fails to converge, that's a bug we want to know about.

**Per-radio TX status (carry-over from v0.3.2):**

| Radio | TX safe to bench? | PureSignal? |
|---|---|---|
| ANAN-G2 / Saturn | **Yes** (primary TX dev target) | **Yes, primary PS target** |
| ANAN-7000DLE / 8000DLE | **Yes** | **Yes** (bench reports especially welcome) |
| ANAN-200D / 100D / 100B / 100 / 10 / 10E | Yes | Yes (bench reports welcome) |
| Hermes / Hermes II / Angelia / Orion / Orion MkII | Yes | Yes (bench reports welcome) |
| **Hermes Lite 2** | **Yes** | **Yes** (HL2-specific path; bench reports especially welcome) |

**Standard items (carried over):**

- ADC Overload badge (yellow on any clip, red on level >3) is your friend
  on TX.
- TUNE button kicks a single-tone carrier.
- VOX is functional; if you don't want VOX, leave it off in the TxApplet
  VOX row before plugging your mic in.
- The new Anti-VOX Gain control on Setup → Transmit → DEXP/VOX should
  silence audible RX-bleed false-trips when you raise it. Leave it at 0
  if you don't have a feedback problem.

## Before you start

### Things you need

- A supported OpenHPSDR radio on your network (any of the table above)
- The platform-appropriate v0.4.0 build from
  <https://github.com/boydsoftprez/NereusSDR/releases/tag/v0.4.0>
- A mic into the radio's mic input (or use the Pc/Composite source via
  VAX) for VOX / Anti-VOX testing
- A wattmeter is ideal for PureSignal calibration verification
- Optional: a second receiver or a friend on frequency for IMD listening

### Get the build

Same download flow as v0.3.2. Replace the version number in the asset
names:

- macOS Apple Silicon: `NereusSDR-0.4.0-macOS-apple-silicon.dmg`
  (signed + notarized)
- macOS Intel: `NereusSDR-0.4.0-macOS-intel.dmg` (signed + notarized)
- Linux x86_64: `NereusSDR-0.4.0-x86_64.AppImage`
- Linux aarch64: `NereusSDR-0.4.0-aarch64.AppImage`
- Windows x64 installer: `NereusSDR-0.4.0-Windows-x64-setup.exe`
- Windows x64 portable: `NereusSDR-0.4.0-Windows-x64-portable.zip`

Verify GPG signatures against `SHA256SUMS.txt.asc` as before.

### Existing settings carry forward

No manual migration step needed. Saved radios, mic profiles, DSP settings,
PA cal-points, per-band tune power, OC matrix, container layout, spectrum
/ waterfall settings carry forward exactly. SettingsSchemaVersion bumps
from v4 to v5 on first launch and migrates the DSP-Options Buffer / Filter
Size keys to per-direction automatically (your existing values stay
intact; the global key just gets split).

## The actual test

### 1. PureSignal end-to-end

This is the headline test for v0.4.0. Plan on 10-20 minutes.

1. Connect to a PureSignal-capable radio (any board with a feedback DDC:
   ANAN-G2, ANAN-7000DLE, ANAN-8000DLE, ANAN-200D, ANAN-100D, ANAN-100,
   Hermes, HL2). Confirm v0.4.0 is running (Help → About).
2. Open **Tools → PureSignal**. The PsForm dialog should open. The layout
   should mirror Thetis PsForm (this is a port, not a NereusSDR-original).
3. Check the bottom banner. The **FB / PS indicator pair** should be
   visible and showing the current state. FB indicates feedback channel
   active; PS indicates correction state.
4. Set the radio to a clean band (20 m or 40 m without QRM is ideal).
5. Pick a TX mode (USB or LSB). Set drive to a moderate level (50% on a
   100 W radio is a good start; 25% on the 8000DLE).
6. Open **Setup → General → Options**. Confirm the PureSignal-related
   checkboxes are wired (HPF Bypass on PS warning, etc.).
7. **Two-tone test**: in PsForm, enable two-tone. The spectrum should
   show two carriers; the IMD overlay should display IM3 and IM5
   markers with peak readouts in dBc. This is your real-time
   linearization view.
8. **Calibration**: in PsForm, run PureSignal calibration (single-cal or
   AutoCal). Watch the AmpView traces (open via PsForm's btnPSAmpView
   button). Convergence typically takes 10-30 seconds.
9. Once converged, key up a real signal (PTT into a dummy load with the
   two-tone disabled). The IMD overlay should show suppressed IM3 / IM5
   compared to pre-correction.
10. **HL2 owners**: the HL2 path uses negative-ATT. If your HL2
    calibration converges, you should see the FB indicator stable and
    the PS indicator showing corrected state. If it gets stuck on
    "calibrating" forever or fails with errors, that's a bug we want.

### 2. The new DSP-Options page

1. Open **Setup → DSP**. The page should populate with the full
   Thetis-faithful 3-column / 4-row layout: 18 controls including
   Buffer Size (RX), Buffer Size (TX), Filter Size (RX), Filter Size
   (TX), filter type combo, filter impulse cache toggles, etc. Warning
   icons should appear next to fields with validity issues; the
   time-to-last-change readout at the bottom should update after each
   change.
2. **In-place filter resize**: change the RX filter size while audio is
   running. The change should apply immediately without dropping audio.
   Same for TX (during MOX or TUNE).
3. **Per-mode live-apply**: switch modes (USB → AM → FM → CW). Each mode
   should pick up its persisted Buffer / Filter / Type values without
   requiring a reconnect.
4. **SettingsSchemaVersion v5 migration**: if you upgraded from v0.3.2,
   on first launch the schema bumps from v4 to v5 and your old global
   Buffer / Filter Size keys split into RX / TX. Verify by checking
   `~/.config/NereusSDR/NereusSDR.settings`; you should see new keys
   like `BufferSizeRX` and `BufferSizeTX` with your old global value
   carried forward.
5. **Filter Impulse Cache**: toggle the cache on. Restart the app.
   Subsequent filter rebuilds should reuse the cached impulse instead
   of recomputing (faster mode/filter switches).

### 3. Spectrum + Multimeter page

1. **FFT slider**: open **Setup → Display → Spectrum**. Confirm the FFT
   slider exposes 7 window types (Rectangular, Hann, Hamming, Blackman,
   Blackman-Harris, Nuttall, Kaiser) and the live bin-width readout
   updates as you change the slider.
2. **NF-aware grid**: enable "Auto-track noise floor" on the same page.
   The grid floor should automatically follow the noise floor as you
   change bands. Per-band NF priming should populate sensible defaults
   on first connect.
3. **Hz/bin auto-zoom override**: change the spectrum zoom; the auto-
   zoom should keep the visual consistency K-based across zoom levels.
4. **SpectrumPeaksPage**: open **Setup → Display → Spectrum Peaks**.
   Enable PeakBlobDetector. Strong signals should get ellipse markers
   with frequency/dBm readouts. Enable Active Peak Hold. The trace
   should display in a distinct colour with per-bin decay.
5. **Multimeter page**: open **Setup → Display → Multimeter**. Confirm
   the page populates. The configurable MeterPoller should drive the
   meter rate; HistoryGraphItem duration should be tunable. MeterItem
   unit-mode should support S / dBm / µV.

### 4. Anti-VOX cancellation feed

This is the test that the slider you couldn't get to do anything in
v0.3.2 now actually does something.

1. Open **Setup → Transmit → DEXP/VOX**. Confirm the **grpAntiVOX**
   group shows three controls: Anti-VOX Enable checkbox, Gain (dB)
   spinbox, Tau (ms) spinbox. All three tooltips should be verbatim
   from Thetis.
2. Enable VOX. Set a VOX threshold above your room noise.
3. With your speakers on (not headphones), put your mic close to a
   speaker. Play some loud RX audio. Pre-v0.4.0 (= v0.3.2 baseline),
   the VOX false-triggers off the speaker bleed.
4. Enable **Anti-VOX**. Slowly raise the **Gain** slider. The
   false-trigger should subside at a level you can hear in real time.
5. Set Tau to 20 ms (default). Try 10 ms and 50 ms; the cancellation
   smoothness should change.
6. **Architectural note**: NereusSDR drops Thetis's source selector
   (RX vs VAC) because VAX in NereusSDR is a digital-mode app bus
   with no mic-feedback path. Setup → Transmit → DEXP/VOX shows a
   static "Source: Audio Output Device(s)" info row instead of a
   selector. If you previously had `AntiVox_Source_VAX` persisted from
   a 3M-3a-iii pre-release, it'll be ignored harmlessly.

### 5. Live-apply sample rate

1. Connect to your radio. Open **Setup → Hardware → Radio Info**.
2. Note the current sample rate combo selection.
3. Change it (e.g., 192 kHz → 384 kHz on a P2 radio, or 96 kHz → 192
   kHz on HL2). The radio should NOT disconnect; the rate should change
   live without an inline reconnect banner.
4. Watch the spectrum and audio: there should be a brief glitch (the
   12-step coordinator drains the channel and reconfigures), but no
   crash and no full reconnect cycle.
5. **Pre-v0.4.0 (= v0.3.2 baseline)**: the same rate change in v0.3.2
   plus PR #221 SIGSEGV'd inside `moveToThread` on the first combo-
   change. v0.4.0 routes through WDSP's SetXcmInrate path instead and
   keeps all wrapper pointers alive.
6. **HL2 P1 384 kHz** (HL2 owners only): the HL2 sample-rate combo
   should now offer 48 / 96 / 192 / **384** kHz. Pre-v0.4.0 the 384 k
   row was missing. mi0bot-Thetis is the authoritative source for HL2.

### 6. AF Gain audio quality + VAX bus calibration

1. **Distortion check**: connect to a strong, clean signal. Set AF
   slider to ~50%. Audio should be clean and clear. Pre-v0.4.0 the AF
   slider was fighting the master-volume atomic post-DSP and the
   panel.gain1=4.0 default was adding a hidden +12 dB; the result
   could be subtly distorted at higher slider positions.
2. **VAX calibration**: open WSJT-X (or FT8 / FLDIGI / Thetis-style
   digital app) on the same machine; route NereusSDR's VAX bus into
   it. Set NereusSDR's VAX channel for your active slice. Receive a
   strong digital signal.
3. With the digital app's audio level meter visible, slide the
   speaker AF slider on NereusSDR up and down. The digital app's
   audio level should NOT change. Pre-v0.4.0 the VAX tap inherited
   the speaker AF attenuation; v0.4.0 inverse-scales by 1/afGain so
   digital apps stay calibrated.
4. **Edge case**: AF=0 (full mute) silences VAX too; full decoupling
   needs a pre-PanelGain1 WDSP tap (deferred). If you mute the
   speaker entirely, expect VAX to go silent.

### 7. Persistence + macOS polish

Restart the app between each step.

1. **MainWindow position / size / maximized state (#206)**: resize
   the window, move it to a secondary display, maximize, restart. All
   three should restore.
2. **Audio mute flush (#201)**: mute the master volume. Wait a
   second. Unmute. Pre-v0.4.0 there was a brief stale-audio replay
   from the speakers ring; v0.4.0 should be silent through the
   transition.
3. **PA Gain spinbox clamp (#199)**: open Setup → PA → PA Gain. The
   gain spinbox minimum should clamp at 38.8 dB (Thetis range).
4. **PA profile auto-pick (#202)**: connect to an ORIONMKII radio.
   The Setup combo should NOT auto-pick ORIONMKII for the model; it
   should default to a non-default model.
5. **Pre-connect Mic_Source persistence**: before connecting, change
   the Mic Source. Quit. Restart. Mic Source should persist across
   restart even before a connection happens.
6. **macOS mic-permission dialog**: on first launch on a fresh macOS
   user, the mic-permission dialog should fire deterministically at
   startup (was sometimes missed before).
7. **macOS app icon**: the bundle icon should be visible in Finder /
   Dock / Spotlight. The DMG should have a custom background and
   volicon.
8. **Step-att MOX (#200)**: set RX step attenuator. Key MOX. Un-key.
   The step-att value should be unchanged. Pre-fix the TX-side
   ATT-on-TX safety scaffolding was over-writing the persisted RX
   ATT.
9. **HL2 FPGA temperature** (HL2 owners): the bottom banner should
   show the HL2 FPGA temperature.

### 8. Regression sweep (carry-forward checks)

These should still work as in v0.3.2. Quick smoke pass:

- Connect via discovery + via unicast probe (VPN-reach).
- TUNE, verify forward power.
- SSB voice TX, verify intelligible audio on a second receiver.
- TX EQ + Leveler + ALC + CFC: load a factory mic profile, hear the
  change.
- TX filter overlay on the panadapter (orange in TX, cyan in RX).
- Per-profile TX bandwidth (TxApplet TX BW spinboxes + status label).
- ADC Overload badge during TX.
- PA voltage on MkII boards reads ~13.x V (regression watch).
- Auto-connect on launch.
- Disconnect overlay (pull Ethernet, see DISCONNECTED fade).
- DEXP / VOX baseline: VOX triggers on voice, releases on silence;
  DEXP gates audio cleanly below threshold; AMSQ on AM modes.
- HL2 RF / Tune Power slider encoding: 0-90 step 6 / 0-99 step 3 /
  per-band Tune in dB.
- PA-cal hotfix: forward power at slider 100% should NOT exceed your
  radio's rated max.

If any of these regressed in v0.4.0, that's a bug.

## Things that might look weird but aren't bugs

- **PureSignal calibration takes 10-30 seconds.** That's normal,
  especially on HL2. Watch the FB / PS indicator pair and the AmpView
  traces.
- **Two-tone test sounds awful on the loudspeaker** but the IMD
  overlay readout is what you want to look at.
- **Active RX Count widget is gone** from Setup → Hardware → Radio
  Info on single-RX builds. The underlying coordinator stays wired and
  the widget will return when Phase 3F multi-panadapter ships.
- **Anti-VOX source selector is gone** from Setup → Transmit →
  DEXP/VOX. Replaced with a static "Source: Audio Output Device(s)"
  info row. Rationale: VAX in NereusSDR is a digital-mode app bus
  with no mic-feedback path, so the audio output device is the only
  valid cancellation reference.
- **PureSignal Setup pages are gone.** Tools → PureSignal opens the
  new PsForm dialog instead. The previous setup-page model was a
  NereusSDR divergence; we never matched Thetis here.
- **Sample-rate change has a brief audio glitch.** That's expected:
  the 12-step coordinator drains the channel and reconfigures the
  AudioEngine + DSP worker. Pre-v0.4.0 it would have crashed.
- **AF=0 (full mute) silences VAX too.** Full decoupling needs a
  pre-PanelGain1 WDSP tap (deferred). For now, don't fully mute the
  speaker if you want VAX audio.

## What we'd love you to try that's unusual

- **PureSignal on every radio we cannot bench locally.** ANAN-7000DLE,
  ANAN-200D, ANAN-100D, ANAN-100, Hermes, Hermes II, Angelia, Orion,
  Orion MkII. Each board has its own peak / sample-rate / DDC
  configuration; the per-board PsDdcConfig should pick the right
  values. **Especially welcome: HL2 PS calibration convergence
  reports.** The HL2 path is new and constrained.
- **Two-tone IMD numbers before/after correction.** The overlay shows
  IM3 and IM5 in dBc. Pre-cal vs post-cal numbers are the most useful
  PureSignal data we can collect.
- **Long PS sessions.** 30+ minute SSB QSOs with PureSignal active.
  Watch for thermal drift, AutoAtt re-converge events, AmpView trace
  drift, FB indicator dropouts.
- **DSP-Options live-apply.** Try to break it: change Buffer Size,
  Filter Size, Filter Type rapidly while audio is running, while
  MOXing, while TUNE is engaged, on a band switch, on a mode switch.
  We want to know the failure modes.
- **Anti-VOX in real-room conditions.** Speakers, room noise, mic
  position. What gain / Tau combination works for your workspace?
- **Sample-rate live-apply on every supported radio.** Cycle through
  the rates available on your hardware. Watch for audio glitches that
  exceed a brief drop, watch for crash, watch for spectrum freeze.
- **VAX calibration with WSJT-X / FT8 / FLDIGI** during real digital
  QSOs. The slider position should not affect the digital app's audio
  level reading.
